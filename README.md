# 实验室样本编号管理系统

基于 Python + SQLite 的实验室样本编号管理工具，支持样本登记、查询、借出、归还及借还记录追踪等完整生命周期管理。

## 功能介绍

- **样本登记**：自动生成唯一样本编号，支持 7 种样本类型，记录样本来源、采集日期、保存位置、负责人及备注
- **样本查询**：支持按样本类型、负责人（模糊）、保存位置（模糊）、状态、采集日期范围进行单条件或组合筛选，支持按登记时间或采集日期排序
- **样本借出**：登记借用人和借出用途，自动校验样本状态，防止重复借出
- **样本归还**：自动更新样本状态和借还记录，防止重复归还
- **借还记录**：支持查询全部或指定样本的借还历史记录
- **命令行交互**：提供交互式菜单，便捷的日常操作界面
- **独立测试数据库**：测试脚本使用独立数据库，不污染正式数据

## 环境要求

- Python 3.7+（推荐 3.10+）
- 标准库：`sqlite3`、`datetime`、`os`、`sys`（无需额外安装第三方依赖）

## 启动方式

### 交互式菜单模式

```bash
python sample_manager.py
```

或在 Windows 上使用 Python Launcher：

```bash
py sample_manager.py
```

启动后按菜单提示进行操作：
1. 登记新样本
2. 查询样本
3. 查看样本详情
4. 样本借出
5. 样本归还
6. 查询借还记录
0. 退出系统

### 作为模块调用

```python
import sample_manager as sm

# 初始化数据库
sm.init_db()

# 登记样本
success, sample_no = sm.add_sample('血液', '张三', '2024-01-15', '冰箱A-01', '李医生')

# 查询样本
samples = sm.query_samples(sample_type='血液', manager='李')

# 借出/归还
sm.borrow_sample(sample_no, '赵研究员', 'PCR实验')
sm.return_sample(sample_no)
```

## 数据库位置

默认数据库文件位于项目根目录：

```
项目目录/
└── samples.db          # 正式数据库（默认）
```

可通过环境变量 `SAMPLE_DB_PATH` 指定自定义路径：

```bash
# Linux/macOS
export SAMPLE_DB_PATH=/path/to/custom.db

# Windows PowerShell
$env:SAMPLE_DB_PATH = "C:\path\to\custom.db"

# Windows CMD
set SAMPLE_DB_PATH=C:\path\to\custom.db
```

相对路径会解析为相对于 `sample_manager.py` 所在目录的路径。

> 测试脚本使用独立的 `test_samples.db`，与正式数据完全隔离，并在测试完成后自动清理。

## 样本编号规则

样本编号格式：`[类型编码][年月][4位序号]`

示例：`BL2026060001`

| 组成部分 | 说明 |
|---------|------|
| 类型编码 | 2 位大写字母，见下表 |
| 年月 | 6 位数字，格式 YYYYMM，为登记时的当前年月 |
| 序号 | 4 位数字，当月同类型样本从 0001 递增，最大 9999 |

**样本类型编码表：**

| 类型名称 | 编码 |
|---------|------|
| 血液 | BL |
| 尿液 | UR |
| 组织 | TS |
| 唾液 | SL |
| 粪便 | FC |
| 脑脊液 | CS |
| 其他 | OT |

编号由系统自动生成，保证全局唯一。当月同类型序号达到 9999 时会返回明确的错误提示。

## 主要操作流程

### 1. 样本登记

```
选择菜单「1. 登记新样本」
  → 选择样本类型编号（1-7）
  → 输入样本来源（必填）
  → 输入采集日期（YYYY-MM-DD，默认今天，必填）
  → 输入保存位置（必填）
  → 输入负责人（必填）
  → 输入备注（可选）
  → 系统自动生成唯一编号并完成登记
```

### 2. 样本查询

```
选择菜单「2. 查询样本」
  → 选择样本类型（0=全部）
  → 输入负责人（模糊匹配，留空不限制）
  → 输入保存位置（模糊匹配，留空不限制）
  → 选择状态（0=全部，1=在库，2=借出）
  → 输入采集日期起止（YYYY-MM-DD，留空不限制）
  → 选择排序方式（登记时间升/降序、采集日期升/降序）
  → 显示匹配结果
```

### 3. 样本借出

```
选择菜单「4. 样本借出」
  → 输入样本编号
  → 系统校验样本存在且状态为「在库」
  → 输入借用人（必填）
  → 输入借出用途（必填）
  → 更新样本状态为「借出」并创建借还记录
```

### 4. 样本归还

```
选择菜单「5. 样本归还」
  → 输入样本编号
  → 系统校验样本存在且状态为「借出」
  → 确认归还（y/n）
  → 更新样本状态为「在库」并填写归还时间
```

### 5. 借还记录查询

```
选择菜单「6. 查询借还记录」
  → 输入样本编号（留空查询全部记录）
  → 显示借还历史列表
```
